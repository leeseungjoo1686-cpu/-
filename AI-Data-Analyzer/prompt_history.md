# 💬 Prompt History (개발 프롬프트 엔지니어링 기록)

본 문서는 기말 프로젝트 평가 기준 중 **"프롬프트 기록을 깃허브 레포지토리에 포함시킬 것"**이라는 조건을 충족하기 위해 작성되었습니다. 웹 애플리케이션의 핵심 로직과 GUI를 구성하기 위해 사용한 프롬프트 이력입니다.

---

## 1단계: RAG 기반 문서 및 링크 분석기 구현 프롬프트

### [프롬프트 1] PDF 텍스트 추출 및 청크 분할 로직 작성
*   **요청 내용**: PDF 문서에서 파이썬 코드로 텍스트를 불러오고, LLM의 토큰 제한에 맞추어 특정 크기로 자르는 가벼운 청크 분할(Splitting) 함수를 설계해 달라고 요청했습니다.
*   **프롬프트 입력문**:
    > "파이썬 `pypdf` 라이브러리를 사용해 업로드된 PDF 파일 전체의 텍스트를 하나의 문자열로 뽑아내는 함수를 짜줘. 그리고 이 텍스트를 토큰 한계를 넘지 않게 약 800자 단위로 쪼개고, 문맥 유실을 막기 위해 150자씩 겹치도록(overlap) 리스트를 만드는 파이썬 함수도 추가해줘."

### [프롬프트 2] Beautifulsoup을 이용한 임의의 웹사이트 URL 본문 긁어오기
*   **요청 내용**: 사용자가 특정 기사나 블로그 URL을 주었을 때 HTML 태그 중 텍스트만 깔끔히 정제하는 크롤러 함수 작성을 요청했습니다.
*   **프롬프트 입력문**:
    > "파이썬 `urllib`로 특정 웹사이트 HTML 소스를 불러와서, `beautifulsoup4` 라이브러리를 통해 script나 style, nav, footer 같은 쓰레기 태그들은 전부 삭제하고 본문 핵심 글자만 정제해서 텍스트로 만들어주는 크롤링 래퍼 함수를 구현해줘."

### [프롬프트 3] RAG 기법 기반 매칭 및 질문 답변 프롬프트
*   **요청 내용**: 사용자의 질문에 맞춰 파싱된 텍스트 청크 중 관계있는 문맥을 필터링해 Gemini API와 결합시키는 핵심 프롬프트 작성을 요청했습니다.
*   **프롬프트 입력문**:
    > "사용자가 질문을 하면 전체 PDF 청크 리스트 중에서 질문에 포함된 키워드가 가장 많이 매칭되는 관련 단락을 검색해 오는 알고리즘을 구현해줘. 그리고 그 단락들을 Context로 묶어 '당신은 AI 튜터입니다. 아래 참고 자료만을 바탕으로 사용자의 질문에 친절한 한국어로 답변해 주세요'라는 프롬프트를 구성해 Gemini 1.5 API에 쏘는 요청 코드를 작성해줘."

---

## 2단계: 실시간 뉴스 감성 분석 대시보드 구현 프롬프트

### [프롬프트 4] 뉴스 RSS 수집 및 파싱 로직
*   **요청 내용**: 복잡한 사이트 크롤링 대신 차단 우려가 적은 구글 뉴스 RSS 피드를 XML 파서로 해석하는 코드를 요청했습니다.
*   **프롬프트 입력문**:
    > "BeautifulSoup에만 의존하지 않고, 파이썬 내장 `xml.etree.ElementTree`를 결합하여 구글 뉴스 검색 RSS 주소(`https://news.google.com/rss/search`)에서 특정 키워드의 기사 제목 10개와 원본 뉴스 링크를 수집해 리스트로 담아주는 파이썬 함수를 만들어줘."

### [프롬프트 5] Gemini API를 활용한 여론 감성 진단 및 JSON 규격화
*   **요청 내용**: AI가 뉴스 목록을 읽고 긍정/부정/중립으로 분류하여 정해진 JSON 딕셔너리로 확실하게 반환하도록 규격 프롬프트를 요청했습니다.
*   **프롬프트 입력문**:
    > "뉴스 기사 목록을 읽어 각 기사가 긍정인지 부정인지 중립인지 AI가 판별하게 할 거야. 에러 없이 화면 차트에 적용하기 위해 결과 데이터를 확실한 JSON 규격으로 받고 싶어. `summary` 필드에는 전체 시장 요약문을, `results` 배열 아래에는 `id`, `sentiment`, `reason` 필드가 들어가도록 Gemini API의 시스템 프롬프트를 설계하고, 마크다운 코드 블록(```json)을 제거하고 딕셔너리로 로드하는 정규식/슬라이싱 파이썬 코드도 구현해줘."

---

## 3단계: 통합 Streamlit 웹 대시보드 UI 구현 프롬프트

### [프롬프트 6] 멀티 탭 및 반응형 대시보드 화면 설계
*   **요청 내용**: 위의 두 프로젝트를 하나의 Streamlit 사이트 형태로 묶고, Plotly 도넛 차트를 활용해 데이터를 시각화하는 전체적인 UI 아키텍처 설계를 요청했습니다.
*   **프롬프트 입력문**:
    > "위에 구현한 RAG 분석기와 뉴스 감성 분석기를 하나의 Streamlit 앱으로 합쳐줘. 상단 탭뷰를 사용해 1번 탭은 RAG 튜터, 2번 탭은 뉴스 분석기가 나오게 해줘. 2번 탭에서는 Plotly 라이브러리를 연동해 긍정/부정/중립 카운트를 바탕으로 세련된 도넛 형태의 파이 차트(Pie Chart)를 그리고, 기사별로 색상이 들어간 배지(badge) 스타일의 HTML 테이블로 상세 결과를 출력하도록 스타일을 묶어서 종합 웹사이트용 `app.py` 단일 파일로 코딩해줘."
